Bully Algorithm (Leader Election)

How it works:

  • Detection: Worker detects leader failure (2 heartbeat timeouts)
  • Election: Send election to all higher-numbered nodes
  • Response: If any respond, they take over; if none respond, I become leader
  • Announcement: New leader announces itself to all peers

Key Benefit: Deterministic - highest ID always becomes leader ✓

Heartbeat Monitoring

Leader Monitors Workers (every 5s)

  • Sends heartbeat RPC to each worker
  • Marks worker dead after 2 failures
  • Routes tasks only to alive workers

Workers Monitor Leader (every 3s)

  • Ping leader with heartbeat request
  • After 2 failures → trigger election
  • Update state when new leader elected

⏱️ Total failure detection time: ~6 seconds (2 failures × 3s interval)

Failure Scenarios Handled

Scenario Detection Result
Worker Dies Leader heartbeat fails ✓ Tasks route to other workers
Leader Dies Worker heartbeat fails ✓ New leader elected, system continues
Multiple Failures Election messages ✓ Highest ID survives, coordinates rest
Network Partition Timeouts ✓ Each partition elects own leader

Failover Scenario

Leader dies → Worker 2 detects failure ↓ Worker 2 initiates election ↓ Worker 2 becomes new leader (highest ID) ↓ New leader announces to peers ↓ Client reconnects → System continues ✓

Testing & Results

Performance

Advantages & Limitations

Advantages ✓

  • Automatic failover without manual intervention
  • Simple deterministic leader election
  • Observable logging for debugging
  • Handles multiple concurrent failures

Limitations ✗

  • Single task type (primes only)
  • No data persistence - registry lost on restart
  • Simple scheduling (no load balancing)
  • Fixed 3-node setup

Future Improvements

  • Persistent State: Use etcd/Consul for distributed consensus
  • Load Balancing: Monitor CPU/memory before task assignment
  • Replication: Replicate leader state to backup nodes
  • Dynamic Scaling: Add/remove nodes without restart
  • Security: TLS encryption and authentication
  • Advanced Algorithms: Raft consensus instead of Bully
  • Monitoring: Prometheus metrics + Grafana dashboards
